Структура робочої директорії

Папка src містить всі вихідні файли проєкту. Вона є основною робочою директорією для розробки і включає в себе логічно розділені підкаталоги, які спрощують організацію коду, його підтримку та масштабування. Всі ресурси, що використовуються на сайті, зберігаються саме в цій структурі. Нижче наведено огляд кожної ключової папки та її призначення.

Директорія html

Папка html призначена для зберігання HTML-розмітки, що є основою всіх сторінок сайту. Для зручності та гнучкості структура розбита на кілька підпапок:

  • components/ — містить багаторазові HTML-компоненти. Це невеликі фрагменти розмітки, які використовуються на кількох сторінках, наприклад: кнопки, картки товарів, елементи списків, модальні вікна тощо.
  • pages/ — тут зберігаються повноцінні сторінки сайту, наприклад index.html, about.html, contacts.html. Кожна сторінка складається з частин, які можуть бути включені через шаблонізатор.
  • partials/ — включає в себе частини шаблонів, які повторюються на всіх або більшості сторінок: header.html, footer.html, navigation.html. Вони дозволяють уникати дублювання коду.
  • images/ — директорія для зображень, які використовуються в HTML (іконки, банери, фотографії, логотипи тощо). Рекомендується зберігати зображення в оптимізованому форматі.

Директорія js

Папка js призначена для JavaScript-коду, який забезпечує інтерактивність і логіку роботи сайту. Тут організовано окремі файли для основних скриптів та допоміжних функцій.

  • functions/ — містить окремі JavaScript-модулі або утиліти, що реалізують певну функціональність (наприклад, слайдери, модальні вікна, таби). Ці функції потім імпортуються у головний файл.
  • app.js — головний файл, у якому виконується ініціалізація основних скриптів. Звідси імпортуються функції з папки functions та викликаються за потребою.

Директорія scss

Папка scss містить усі стилі проєкту, написані на препроцесорі SCSS. Для підтримки порядку та масштабованості структура стилів поділена на логічні блоки:

  • components/ — стилі для окремих UI-компонентів, таких як кнопки, форми, модальні вікна, таби тощо. Кожен компонент має власний SCSS-файл.
  • pages/ — стилі, що стосуються конкретних сторінок сайту. Це дозволяє чітко відокремити загальний стиль від специфічного.
  • partials/ — допоміжні файли стилів, які використовуються в інших SCSS-файлах. Наприклад, стилі для базових типографічних елементів, флекс-утиліт, а також layout-блоків.
  • _mixins.scss — файл з міксинами SCSS, які полегшують повторне використання коду. Наприклад, адаптивні відступи, медіа-запити, обгортки з flex/grid.
  • _variables.scss — містить усі змінні проєкту: кольори, типографіку, брейкпоінти, шрифти тощо. Завдяки змінним проєкт легко підтримувати і змінювати візуально.
  • main.scss — головний SCSS-файл, у якому підключаються всі попередні стилі через @import або @use. Саме цей файл компілюється у CSS.

Директорія fonts

За потреби ви можете створити папку fonts в директорії src для зберігання шрифтів, які використовуються у проєкті. В цю папку можна додавати файли шрифтів у форматах .woff, .woff2, .ttf тощо. Під час збірки ця папка автоматично буде скопійована в фінальну директорію dist, зберігаючи структуру та вміст.

Рекомендується використовувати оптимізовані формати (наприклад, .woff2) для кращої продуктивності сайту.
