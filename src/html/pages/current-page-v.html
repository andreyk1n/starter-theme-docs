Визначення поточної сторінки

Цей функціонал призначений для автоматичного визначення, яка саме сторінка зараз відкрита, і підсвічування відповідного пункту меню в навігації. Коли користувач переходить на будь-яку сторінку сайту, скрипт спочатку отримує поточний шлях сторінки за допомогою window.location.pathname. Цей шлях може виглядати, наприклад, як /about.html. Метод split('/') розбиває цей шлях на частини, а .pop() бере останній елемент — тобто назву файлу сторінки. Якщо ж ім’я файлу відсутнє, як у випадку головної сторінки (де шлях буде просто /), тоді підставляється 'index.html' за замовчуванням.

Далі скрипт перебирає всі елементи з класом .nav__link — це посилання в головному меню. Для кожного такого посилання він порівнює значення атрибута href з назвою поточної сторінки, яку ми тільки-но визначили. Якщо значення збігаються, тобто якщо href посилання співпадає з тим, яка сторінка зараз відкрита, скрипт додає до цього посилання клас currentPage.

JavaScript-логіка


const currentPath = window.location.pathname.split('/').pop() || 'index.html';
document.querySelectorAll('.nav__link').forEach(link => {
    if (link.getAttribute('href') === currentPath) link.classList.add('currentPage');
});

Стилізація


.currentPage {
  font-weight: bold;
  color: #ff6600;
}

Цей клас можна використовувати в CSS для стилізації активного пункту меню, наприклад, зробити його жирним, змінити колір або підкреслити. Таким чином, користувач завжди буде бачити, на якій сторінці він знаходиться, і не потрібно вручну додавати спеціальний клас до кожного посилання на всіх сторінках сайту. Усе відбувається автоматично при завантаженні сторінки.

Рекомендації

  • Підтримка підкаталогів: якщо сайт має структуру з підкаталогами (наприклад, /blog/article.html ), поточна логіка може не працювати коректно. У такому разі варто враховувати повний шлях або частину шляху.
  • Порівняння з location.href: у складніших випадках, коли href містить абсолютні або відносні шляхи, варто привести href до повного вигляду перед порівнянням або використовувати new URL().
  • Плавна анімація активного пункту: можна додати ефекти типу transition, щоб зміна стилю була плавною.

  • SEO-дружність: підсвічування активного посилання може позитивно впливати на користувацький досвід і знижувати показник відмов, що важливо для SEO.
