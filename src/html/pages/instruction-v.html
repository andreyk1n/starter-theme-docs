Інструкція з ініціалізації та розгортання проєкту

Ця сторінка містить покрокову інструкцію для запуску та розгортання вашого фронтенд-проєкту, що базується на Gulp. Ви дізнаєтесь, як ініціалізувати тему локально та як задеплоїти готову версію сайту на GitHub Pages.

Ініціалізація теми

Після завантаження репозиторію з темою, виконайте наступні дії для початку роботи з проєктом:

Крок 1: Встановіть залежності

Відкрийте термінал у кореневій папці проєкту та виконайте команду:


npm install

Це встановить усі необхідні залежності, вказані у файлі package.json, включно з Gulp та плагінами, які використовуються в проєкті.

Крок 2: Запустіть Gulp

Після завершення встановлення залежностей, запустіть Gulp для компіляції файлів та запуску локального сервера:


gulp

У результаті запуститься середовище розробки з автоматичним оновленням сторінки при зміні файлів (Live Reload). Зазвичай воно буде доступне за адресою http://localhost:3000/.

Крок 3: Розгорніть через Git Subtree

Щоб розгорнути ваш проєкт, використовуйте команду git subtree, яка дозволяє пушити підкаталог (dist) як окрему гілку:


git subtree push --prefix dist origin gh-pages
Цю команду слід виконувати кожного разу після внесення змін, щоб оновити сайт на GitHub Pages. Вона автоматично витягує вміст папки dist і пушить його у гілку gh-pages.

Крок 4: Налаштуйте GitHub Pages

  1. Відкрийте ваш репозиторій на GitHub.
  2. Перейдіть у розділ Settings → Pages.
  3. У полі Source оберіть гілку gh-pages і підтвердьте вибір.
  4. Після збереження налаштувань GitHub почне процес публікації.

Крок 5: Перевірте результат

Зазвичай через 1–2 хвилини ваш сайт буде доступний за адресою:


https://ваше_ім’я_користувача.github.io/назва_репозиторію/

Архівація проєкту

Для збереження резервних копій або передавання проєкту стороннім особам рекомендовано виконувати архівацію. Ви можете архівувати:

  • лише зібрану папку dist (готову до деплою)
  • або весь проєкт за винятком службових файлів і директорій

Для цього в проєкті реалізовано два асинхронні завдання, які можна запускати через Gulp або окремо:

Архівація папки dist

Функція zipDist() створює ZIP-архів поточної збірки. Назва архіву автоматично включає дату, а також індекс, якщо архів з такою назвою вже існує


gulp zipDist

Архівація всього проєкту

Функція zipProject() архівує весь проєкт, ігноруючи службові папки (node_modules, .git, archives) та тимчасові файли:


gulp zipProject

Це корисно для зберігання повної копії робочої версії проєкту або під час передачі іншому розробнику.

Рекомендації

  • Якщо ви регулярно оновлюєте сайт, створіть alias або npm-скрипт для команди git subtree push.
  • Завжди перевіряйте вміст папки dist перед публікацією — уникайте випадкових файлів, що можуть туди потрапити.
  • Уникайте ручного редагування гілки gh-pages, щоб уникнути конфліктів — усі зміни мають йти лише через автоматизовану збірку.
