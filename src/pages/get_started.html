Інструкція з ініціалізації та розгортання

Ініціалізація теми

Для початку роботи з темою виконайте наступні команди: 

            
npm install
gulp

Розгортання проєкту на GitHub Pages

Використовуйте ці кроки для розгортання вашого проєкту з Gulp на GitHub Pages:

  1. Зібрати проєкт локально: Виконайте всі команди для збірки (наприклад, gulp). Це створить статичну версію проєкту у папці для результатів (зазвичай dist або build).
  2. Підготуйте dist або build для розгортання: Після збірки проєкту переконайтеся, що GitHub Pages налаштовано для розгортання з цієї папки.
  3. Створіть гілку gh-pages: Для розгортання тільки з папки dist, створіть нову гілку gh-pages та завантажте туди лише вміст цієї папки: 
                    
git subtree push --prefix dist origin gh-pages
    Цю команду потрібно писати кожного разу після того як ми запушили зміни на основну гілку, тоді вони будуть підтягуватись з неї, і за декілька хвилин задеплоються на Githu Pages
  4. Налаштуйте GitHub Pages: Вкажіть GitHub використовувати гілку gh-pages для розгортання:
    • Відкрийте налаштування вашого репозиторію на GitHub.
    • У розділі "GitHub Pages" оберіть gh-pages як основну гілку для розгортання.
  5. Перевірте розгортання: Через кілька хвилин ваш сайт стане доступним за URL: https://<ваше_ім'я_користувача>.github.io/<назва_репозиторію>.
